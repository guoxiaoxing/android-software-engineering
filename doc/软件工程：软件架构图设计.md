类：软件系统的最小结构单元。

组件：组件是一个或者多个类组成的逻辑群组，用来完成某个逻辑功能，组件通常由多个类在更高层次的约束下组合而成。

容器：容器是一个在其内部可以执行组件或者驻留数据的东西，

系统：系统是最高的抽象层次，代表了能够提供价值的东西。一个系统由多个独立的容器构成。

针对这四个层次，我们用四种图来阐述它们。

语境图：设定场景的高层次图，包括关键的系统依赖于参与者。

容器图：容器图显示了高层次的技术选择，容器如何分担职责，如何通信。

组件图：组件图可以让我们看到每个容器的关键逻辑组件以及它们之间的关系。

类图：类图是可选的，它被用来解释某个组件如何实现。


## 语境图

**目标**

```
1 我们构建的软件系统是什么？
2 谁会去使用它？
3 如果兼容已有的环境？
```

**目标**

在中间画一个简单的框图展示你的系统，周围是它的用户和其他与之相互作用的系统。

## 容器图

**目标**

```
1 软件系统的整体形态谁什么样的？
2 高层次技术决策有哪些？
3 职责在系统中如何分配？
4 容器之间如何通信？
```

每个容器都可以指定：

- 名称：容器的逻辑名称
- 技术：容器实现的技术方案
- 职责：容器职责的高层次声明或者清单

## 组件图

**目标**

```
1 容器有哪些组件构成？
2 在高层次上，系统如何工作是否清晰？
3 所有组件是否都驻留在一个容器中？
```
